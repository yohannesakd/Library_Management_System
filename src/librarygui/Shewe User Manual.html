Overview

Shewe Library Management System is aimed to substitute the existing classical and wrecked paper-based library system of a certain Library enabling the library members to perform all the specified duties assigned to a library member in most effective manner.

 

Library management systems help libraries keep track of their documents inventory and loans, and member subscriptions and profiles, sometimes for multiple physical locations. This type of software can be used by educational institutions’ libraries, as well as public or private libraries. Library patrons use library management systems to find, reserve, and loan documents while library staff employ it to manage the acquisition, cataloging, and inventory of documents.

 

Implementation of this project enables the library staff (librarian) to make entry of new library resources i.e. books, members, and etc. As well as updating, deleting and issuing a book to a member, find out details of library resources such as total number of books and members accurately for example, members by their name, address, and books which are issued to members and did not return on time and many more.

 

The library members are able to use this project to find out details of all the books which are present currently in the library and determining whether they are exist inside the library or borrowed by library members and the due time for returning the borrowed book and etc.

 

User Roles and Definition

There are different possible roles for accessing and use this system which are as follows:

1. Member: The privilege of a member is at he/she is able to search current available books in the library. Only registered members can issue books and members does not need username and  password to access the system. An instance of the program that just displays the current available books is done so that member can view which book is available for them.

 

2. Librarian: Users with privilege of librarians could have access to the most features of the system than the member and are authorized to manage the books which are being issued and being returned and add members of the library.

 

3. Administrator: While you enter the system or have access to the system with the privilege of the administrator you are able to manipulate all of the library resources including those that are managed by the librarian and additional features such as adding or removing books, promoting or demoting admins from their librarian position and so on.

 

Basic Features of the System

1. Profile section: as a user (admin or librarian) you can access your account information and edit or delete some of your relevant information like changing your password, email and etc.

 

2. Service section: This section refers to the library administrators through which they are able to manage their own resources i.e. library members, books, depositing library resources to the members, customizing the library information, email and etc. Last but not least can view list of issues in which users whose time for returning the borrowed books has expired and they haven’t returned the book.

 

3. Master section: This is also a section referred to the administrators of the library that enables them to manage the master information like: book authors, publishers, categories, and etc… which are needed for identifying the books and library members.

 

4. Statistic and Summery: This section is one of the most useful parts of this system that provides various useful information about library resources i.e. books, members, issues and etc…

 

User Manual

1. Logging In

 

 

 

Log in into the system by choosing between entering as an Admin or Librarian and provide your username and password.

 

 

2. Entering as an Admin

 

 

This is the default home page when you log in as an admin the left side area lets you choose among which lists you would like to see and the center part is a table view of those lists.

 

2.1. Home Tab

The Home Tab is just an introductory page that displays the statistics of the library.

 

2.2. Books Tab

 

 

In the Books tab, you can see the current lists of book, and you can modify them by the buttons found at the right side of the screen. Some of the operations are:

 

2.2.1. Searching Books

 

Here you can search for books with whatever field you like such as book title, author, category etc.

2.2.2. Adding Books

 

By clicking the “Add Book” button you can add a book to the system by filling out the required fields as shown above. Any error that may arise during this action is handled by custom exception classes as well as built-in ones.

 

2.2.3. Editing Books

 

Before editing books make sure you have selected the book you want to edit. Then Click the “Edit Book” button.

 

After selecting the book you can edit the fields you wanted to change.

 

2.2.4. Removing Books

 

Before removing books, again, make sure you have selected the book you want to delete from the list. Then you can press the “Remove Book” button.

 

2.3. Issues Tab

 

Issues Tab has similar interface with the Home Tab, we have the issues list table at the center and other buttons to navigate through other tabs from the left side and additional operations to be performed regarding book issues on the right side.

 

2.3.1. Searching for Issues

 

You can for issues with whatever fields you like such as member name, book title or issue date.

2.3.2. Making New Issue

 

You can create new issue by clicking “New Issue” and an issue form will appear.

 

You can create an issue by providing Member ID, Book ID and the Date to be returned from the date picker.

 

 

After inserting necessary information you can click the “Confirm Issue” button and the issue will be saved and will be displayed on the Issue List table.

 

2.3.3. Returning Books

 

To confirm the return of a book click the “Book Return” button.

 

After clicking the “Book Return” button a form will appear that will prompt you to enter the Member ID that issued the book and the Book ID that was issued.

 

After inserting the Book ID and Member ID details about the issued is displayed then you can click the “Confirm Return” button in order to confirm that the book has been returned.

2.3.4. Adding Members

 

 

 

In case, the member that is issuing the book is not the member of the system the Admin can add the member with out further ado just by clicking the “New Customer” button at the right bottom of the screen.

 

After clicking the “New Customer” button the Admin can add the member with this pop-up window.

 

2.4. Customers Tab

 

Again Customers Tab has similar interface with the Home Tab, we have the members/customers list table at the center and other buttons to navigate through other tabs from the left side and additional operations to be performed regarding member addition and deletion on the right side.

 

2.4.1. Searching Customers/Members

 

You can search members with whatever field you want.

2.4.2. Adding Members

 

To add members click the “Add” button.

 

After clicking the “Add” button the Admin can add the member with this pop-up window.

2.4.3. Delete Members

 

Before deleting a member make sure that the member to be deleted is selected.

 

Then you can click the “Delete” button to remove the member.

 

2.5. Librarians Tab

 

Again Librarians Tab has similar interface with the Home Tab, we have the librarians list table at the center and other buttons to navigate through other tabs from the left side and additional operations to be performed regarding librarian addition and deletion on the right side.

 

2.5.1. Searching for Librarians

 

You can search librarians with whatever field you want from the search field.

 

2.5.2. Add Librarian

 

To add a librarian click the “Add Librarian” button.

 

After clicking the “Add Librarian” button you can fill the fields and click the “Register” button to add the librarian.

 

2.5.3. Removing Librarian

 

Before removing a librarian make sure you select the librarian you want to remove.

 

Then click the “Remove Librarian” button.

 

2.6. Administrators Tab

 

Again Librarians Tab has similar interface with the Home Tab, we have the librarians list table at the center and other buttons to navigate through other tabs from the left side and additional operations to be performed regarding librarian addition and deletion on the right side. However, the “Administrators” button is found at the bottom, unlike the other buttons, since administrators have special privileges in the system.

 

2.6.1. Promoting an Admin

Admins are promoted from librarians list. Hence, in order to be an admin, the admin to be promoted must be present on the librarian list.

 

In order to promote an admin, insert the librarian id then click “OK”. Newly promoted admins will have default password which is “admin”. They can change it later by going to the menu bar and select the “Edit Profile” option. See how to edit profile later in this section.

 

2.6.2. Demote an Admin

Demoting an admin simply removes the user from admin privileges and returns he/she back as a librarian.

 

Before demoting an admin make sure you select the admin to be demoted.

 

Then click the “Demote Administrator” button.

 

2.6.3. Removing an Admin

Removing an admin completely removes the user from the system by deleting he/she from both librarian and admin lists.

 

Before removing an admin make sure it is you select the admin to be removed.

 

Then click the “Remove Administrator” button.

 

2.7. Menu Bars

 

The Menu Bars in the program has two tabs: “Menu” and “Help”. Under the “Menu” tab you can edit your profile, log out and exit from the program. While on the “Help” tab the user can find the “About” and “User Manual” pages.

 

2.7.1. Editing Profile

 

To edit profile go to the “Menu” tab from the menu bar and click “Edit Profile”

 

Then, a pop-up window appears and you can edit your fields.

2.7.2. Logging Out

 

To log out go to the “Menu” tab from the menu bar and click “Log Out”.

 

3. Entering as a Librarian

 

To enter as a librarian select the “Librarian” choice and enter your username and password.

 

 

The Librarian interface is similar to the Administrator one except they have less privileges. Librarians can only issue books, confirm returned books and add members.

 

The operations done by the librarian is similar with the admins one, so you can refer to the lists above.
